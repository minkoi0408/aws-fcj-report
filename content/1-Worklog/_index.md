**In this section** you will need to introduce your worklog, **how** it is structured, **how many weeks** the program lasted, and **what you did** during those weeks.

Typically, and as a standard, a worklog is completed over approximately 3 months (during the internship period) with the following content for each week:

**Week 1:** [Familiarization with AWS and basic services in AWS](1.1-week1/)

**Week 2:** [Deploying EC2, practicing IAM Roles and advanced AWS CLI](1.2-week2/)

**Week 3:** [Mastering VPC, Subnets, Internet Gateway, and SG/NACL security](1.3-week3/)

**Week 4:** [Exploring Amazon S3: Storage Classes, Versioning, and Static Website Hosting](1.4-week4/)

**Week 5:** [Deploying Amazon RDS, configuring Multi-AZ and Read Replicas](1.5-week5/)

**Week 6:** [Working with DynamoDB (NoSQL) and ElastiCache (Caching Layer)](1.6-week6/)

**Week 7:** [Configuring Auto Scaling Group (ASG), ALB, and monitoring with CloudWatch](1.7-week7/)

**Week 8:** [Building Serverless architecture with AWS Lambda and API Gateway](1.8-week8/)

**Week 9:** [Handling Data Streaming with Amazon Kinesis and ETL pipeline with AWS Glue](1.9-week9/)

**Week 10:** [Integrating asynchronous applications using SQS, SNS, and Step Functions](1.10-week10/)

**Week 11:** [Implementing advanced security: AWS WAF, Shield, and GuardDuty](1.11-week11/)

**Week 12:** [Reviewing all knowledge, resource cleanup, and certification guidance](1.12-week12/)
